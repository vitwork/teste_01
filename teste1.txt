Notas Git

Iniciar o git pela primeira x:
git —- version


Iniciar um repo (mkdir e dir para lá antes, obviamente)
git init

Adicionar files (sem commit nem nada, estão lá simplesmente)
git touch [file.txt]

Listar as files no dir
ls

Para o git dizer as files que lá estão e em que situação
git status

Para preparar o commit, tem e haver files. Primeiro:
git add [file]

Noção: adiciona as files ao “staging environment” ou “staging” (ou ainda index, que era o nome antigo, hoje a evitar). 

Se se fizer o git status agora, ele vai indicar as files adicionadas mas atenção: ad não foram commited! Isso só acontece qd se faz o commit.

Depois de adicionar td o que queremos, pode-se então criar o commit, um registo instantâneo de tudo:
git commit -m "Your message about the commit"

Faz o commit e indica td o que se alterou dd o último commit.
A mensagem a pôr deve ser alusiva às alterações, mas atenção: é uma msg que vai ser lida por todos, por terceiros, logo não deve ser estupida.
